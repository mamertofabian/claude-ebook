# CLAUDE.md — claude-ebook

Build system + collection that compiles Anthropic's official Claude docs, cookbooks, courses,
and engineering guidance into a single cohesive offline ebook (**EPUB + AZW3**).

## What exists today (the real pipeline)

The blueprint is now implemented. The flow is:

```
fetch_docs.py  +  notebooks (nbconvert)   ->   build_book.py   ->   pandoc EPUB   ->   calibre AZW3
   (web docs)        (repo .ipynb -> .md)      (build/book.md)      (output/*.epub)    (output/*.azw3)
```

Real code/artifacts:

- `anthropic_resources.py` — resource harvester → link index `anthropic_resources.{md,json,csv}`
  (~960 links; Claude Code pages now included). Stdlib-only, Python 3.8+.
- `fetch_docs.py` — fetches the web-only content (platform.claude.com `.md`, code.claude.com `.md`,
  7 engineering essays HTML→md) into `docs/`. Reuses `fetch()`/`parse_llms()` from the harvester.
  Fail-loud: missing/short pages are collected and the run exits non-zero.
- `build_book.py` — the assembler. Reads `manifest.txt`, demotes headings (Part = H1, each source = H2),
  rewrites relative image paths, cleans MDX (frontmatter + JSX), adds `*Source:*` lines → `build/book.md`.
  Pure transforms are unit-tested; I/O is at the edges.
- `manifest.txt` — the curated reading order (8 Parts). **Single source of truth for curation.**
- `frontmatter/` — authored preface + per-Part intros (inserted verbatim via the `~ ` directive).
- `metadata.yaml`, `Makefile` — book metadata and one-command builds.
- `test_*.py` — `unittest` suites (`make test`).
- `anthropic-collection/` — 7 cloned official repos (~666 MB). **Git-ignored** — never `git add` it.
- `docs/` — fetched page content (generated by `fetch_docs.py`). Git-ignored build input.

## Commands

```bash
make fetch       # download web-only docs into docs/
make notebooks   # convert the curated .ipynb set to .md (uvx nbconvert)
make all         # assemble book.md -> EPUB + AZW3 in output/
make test        # run all unittest suites
python3 anthropic_resources.py --no-api-ref   # rebuild the link index
```

## Gotchas

- **Pandoc 2.x** is installed (`2.9.x`). The Makefile auto-selects `--epub-chapter-level=1`
  (the 3.x flag is `--split-level=1`). Don't hardcode the 3.x flag.
- **PDF is intentionally not built** — no TeX dependency. EPUB + AZW3 only.
- **The EPUB is built with `pandoc -f gfm`, not the default reader.** Pandoc 2.x's default
  `markdown` reader hits exponential backtracking on the ~6 MB book (it ballooned to 54 GB RSS
  and never finished); `-f gfm` (linear cmark parser) builds it in ~1 minute at ~1 GB. The
  Makefile already does this — never drop `-f gfm`. `metadata.yaml` is passed via
  `--metadata-file` so gfm doesn't parse it as content.
- **MCP docs are `.mdx`** (JSX), and the harvester only counts `.md`/`.ipynb`, so it under-reports MCP.
  `build_book.py` cleans `.mdx` on the way in (frontmatter + JSX stripped).
- **Curation lives in `manifest.txt`**, not in code. To add/remove/reorder content, edit the manifest;
  it fails loud on any missing path. Cookbook dirs mix notebook-derived `.md` with repo scaffolding
  (`README.md`, `skill.md`, output reports) — the manifest lists only the notebook recipes.
- **Refresh `anthropic-collection/` with `git pull`** — it's intentionally ignored, never re-committed.
- Per global setup, use `uvx nbconvert` rather than `pip install nbconvert`.
